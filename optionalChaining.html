Optional Chaining

Optional chaining (?.) is similar to chaining operator(.), it accesses the object property or call a function. But if the object property does not exist(undefined or null) then then the expression short circuits and evaluates to undefined instead of throwing an error.

when using chaining operator it throws an error if the object property or function is undefined or null. So optional chaining checkes whether the property or function called is defined or not, and if not then the undefined is returned and the execution continues.
